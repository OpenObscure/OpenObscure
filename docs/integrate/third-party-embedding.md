# Embedding Overview

OpenObscure can be compiled directly into an iOS, Android, or macOS application as a native library. In this **embedded** model, PII detection and encryption happen in-process â€” no HTTP proxy, no sidecar, no network hop. Your app calls `sanitizeText()` before sending user input to an LLM and `restoreText()` when the response arrives. The library ships as a static `.a` (Apple) or shared `.so` (Android), with Swift and Kotlin bindings generated by UniFFI.

This is the right choice when you control the app's source code and want the lowest-latency, most private integration. If you don't control the app â€” or it already talks to an LLM over HTTP â€” use the [gateway proxy](../get-started/gateway-quick-start.md) instead, which requires zero code changes (just a base URL swap).

| App | Platform | Model | Build Status | Approach |

### Integration Artifacts

| Artifact | Enchanted (macOS) | RikkaHub (Android) |
|----------|-------------------|-------------------|
| Native library | 158MB static `.a` (macOS) / 160MB (iOS) | 24MB `.so` (arm64-v8a) |
| Bindings | `openobscure_core.swift` + FFI header + modulemap | `openobscure_core.kt` (UniFFI) |
| Key storage | iOS Keychain | Android SharedPreferences âš  see note |
| Intercept pattern | Direct API calls in ConversationStore | OkHttp Interceptor on request JSON |

> **âš  Android key storage:** The RikkaHub reference integration uses `SharedPreferences`, which stores data in plaintext on the device filesystem. For production use, replace this with `EncryptedSharedPreferences` backed by the Android Keystore. Plain `SharedPreferences` is acceptable for development and testing only.

## Build Scripts Reference

| Script | What it builds | Output |
|--------|---------------|--------|
| `build/build_ios.sh` | iOS static libs + XCFramework | `.a` files + `.xcframework` |
| `build/build_android.sh` | Android shared libs | `.so` per ABI |
| `build/generate_bindings.sh` | UniFFI Swift + Kotlin bindings | `bindings/swift/`, `bindings/kotlin/` |
| `build/bundle_models.sh` | Copy & rename models for embedded apps | `<output_dir>/{ner,scrfd,ocr,nsfw,...}/` |
| `build/download_models.sh` | ONNX model files | `openobscure-core/models/` |
